Final Project –               100 Possible Points (30% of Grade) 

Present your Final Project at the Student Showcase/Classroom 

 

Rubrics for Final Project Grading - OOP and Design Patterns Final Project Rubric.docx 

Requirements

  1. Form a team of 2 members. 

  1. Set up a public GitHub to collaborate as a team; add all your members as a collaborators 

  1. Add project repository <username/repo-name> in this file: OOP-Final-RepositoryList.docx 

  1. Abstract (max 200 words) along with a project title due to me at least 2 days before the due date of application. 

  1. Add your abstract into the repository by the due date so that I can proofread your abstract and okay it before the official showcase application due date 

  1. Use ChatGPT if needed to polish the writing. 

  1. Apply for the Student Showcase application by submitting team info along with the abstract 

  1. Create 4+1 Views of your software design analysis 

  1. The group must provide an update on their progress during a weekly standup in the class 

  1. Present the application at the Student Showcase (https://www.coloradomesa.edu/showcase/index.html) 

  1. Mostly focus on technical aspects (see technical requirements) of the app and not so much on the functional requirements 

  1. Get grades, win bragging rights, and build a resume 

Project technical requirements (**MUST**) 

  1. Must use Object-Oriented Principles and Design  

  1. Each team member must contribute equally for the successful execution of the project; there’s no team grade; but only individual grade that depends on individual contributions 

  1. Each member must write at least two modules (one for a Class and 1 for a test module)  

  1. Each member must write at least one Test module for their team member’s Class definition 

  1. Each member must create and push their own branches adhering to the best naming convention tying to the issue(s)  

  1. Must use at least 3 design patterns (see textbook or https://refactoring.guru/design-patterns/ )  

  • Explicitly discuss/present the design patterns you’ve picked and why; why they’re useful in solving your problem  

  1. Must use Docker (https://github.com/rambasnet/course-container) , CI-CD, Makefile, mypy, Python style guide, git best practices, etc. like in individual assignment 

  1. Must use unit testing (mocking/patching) and property-based testing using hypothesis  

  1. Must produce design documentation of the product such as UML diagrams, API and code documentation, etc.  

  1. Must use a software management tool (e.g., GitHub Project, Clickup, MS Teams, etc.)  

  1. Must document each member’s responsibility for various parts of the projects  

  1. can use Gant chart, kanban board, etc.  

Project Ideas 

  • Use any of CLI, GUI, Web-based interfaces 

  • Simple games such as Tic Tac Toe, Hangman, Connect Four, Chess, Memory puzzle, sliding puzzle, Maze, etc. 

  • Machine learning applications: an app using ML, ChatGPT API, deep learning app in computer vision, etc. 

  • Business applications - inventory management system, online shopping, banking, etc.  

Final Deliverables

  • Add project repository <username/repo-name> in this file: OOP-Final-RepositoryList.docx. Make sure the repo is public. 

  • Add your presentation to the "grades" folder along with the pictures of judges’ grades (if any). 

  • As a team, self-grade your project using the provided Rubric (must justify your grade in the comments), convert it to a PDF, and upload it to the grades folder.

  • Update your README with project and team information, judges’ average score, self-grade, etc. 

  • Include “How to Run” and “How to Test” sections with detailed steps, including screenshots and/or giffies if necessary. 

  • Use the Rubric for self-grading – judge your own work as a team. Provide a brief justification of your grade. 

  • Provide screenshots of any third-party software management tools used for the project as evidence. 

  • Create a screenshots folder and save all the screenshots in that folder -- more the merrier as the proof of completion, testing (manual and automation), etc. 

  • Do a final add, commit, and push!
